# Critic 升级决策

**决策日期**: D8+1
**依据**: A/B blind test 结果 + Phase 3 稳定性数据

---

## 决策: **暂不升级 LLM Critic，保留规则 Critic**

---

## 理由

### 1. 当前规则 Critic 的价值已充分兑现

| 能力 | 状态 | 证据 |
|------|------|------|
| Taboo 安全拦截 | **100%** | 0/23 漏判 (D2-D8) |
| L1-L5 维度评分 | **稳定** | pass_rate 恒定 90% |
| 成本控制 | **$0.0132/task** | 远低于 $0.08 门槛 |
| 候选选择 | **无增益** | A/B tie 18/18 |

### 2. LLM Critic 升级的 ROI 分析

| 项目 | 当前 (规则) | 升级后 (LLM) | 增量 |
|------|-----------|-------------|------|
| 候选选择质量 | 随机 (tie) | 可能区分 | **不确定** |
| 成本/task | $0.013 | ~$0.05-0.10 | **+3-8×** |
| 延迟 | 5-6s | 10-20s | **+2-3×** |
| Taboo 安全 | 100% | 100% | **0** |
| 复杂度 | 零依赖 | 需 LLM API | **高** |

### 3. 升级条件 (未满足)

以下条件都不成立，因此不升级：

- [ ] pass_rate < 85% → 当前 90% **稳定**
- [ ] taboo 漏判 > 0 → 当前 0/23 **完美**
- [ ] 用户反馈图片质量差 → **无此反馈** (prototype 阶段)
- [ ] 候选间质量差异大 → A/B 显示 **无差异** (规则 Critic 无法感知)
- [ ] 有预算承担 LLM 调用 → **$5 剩余**, 需保留给图片生成

---

## 保留策略

### 短期 (当前 prototype)

1. **保持规则 Critic** — taboo 安全 + L1-L5 评分已满足需求
2. **Taboo 检测** — 唯一不可替代的 Agent 价值，必须保留
3. **Scout 增强** — 如需提升质量，优先增强 prompt 而非评分

### 中期 (如果进入生产)

当以下条件出现时，考虑 LLM Critic：
1. 有足够 API 预算 (>$50/月)
2. 用户反馈需要更精准的图片质量评估
3. 候选生成质量提升到规则 Critic 无法鉴别的水平

### 推荐的渐进路线

```
Phase 1 (当前): 规则 Critic (taboo + L1-L5)     ← 已完成
Phase 2 (可选): 规则 Critic + 轻量 VLM 校验      ← 仅对 taboo 边界案例
Phase 3 (远期): 全量 LLM Critic                   ← 当预算和需求匹配时
```

## 全程数据支撑

| 指标 | D2-D8 全程 | 门槛 | 结果 |
|------|-----------|------|------|
| 总 API tasks | 320 | — | — |
| Pass Rate | 90.0% (恒定) | ≥85% | **超标** |
| Taboo 漏判 | 0/23 | =0 | **完美** |
| Avg Cost | $0.0132 | ≤$0.08 | **远低** |
| P95 Latency | 10720ms | <30s | **远低** |
| Fallback 触发 | 0 | ≤5% | **零触发** |
| 候选选择增益 (A/B) | 0% | — | **无增益** |

---

## 结论

**规则 Critic 在当前 prototype 阶段已达最优 ROI。** 升级 LLM Critic 的预期增益不足以抵消成本、延迟和复杂度的增加。Taboo 安全拦截是 Agent pipeline 的核心价值，且已 100% 可靠。
